# Study Notes: Patch-Aware Anomaly Detection for TyreVisionX

## Big-Picture Idea
TyreVisionX is trying to find defective tyres by learning what normal tyres look like. The current strongest anomaly candidate uses one pooled ResNet50 embedding per image. That is simple and strong, but it can still miss small local defects because the whole image is compressed into one vector.

Patch-aware anomaly detection asks a more local question:

> Does every small region of this tyre look like some normal tyre region seen during training?

This is closer to how a human inspector works. A human does not average the whole tyre into one impression. They scan local areas for cracks, cuts, bulges, strange texture, and edge defects.

## Convolutional Feature Maps
A convolutional neural network, such as ResNet50, processes an image through layers. Before the final pooled embedding, the network produces feature maps. A feature map is a grid of vectors. Each grid cell corresponds to a different spatial location in the image.

For example, a ResNet50 layer can output a tensor shaped like:

```text
channels x height x width
```

If the output is `2048 x 7 x 7`, then there are 49 spatial positions, and each position has a 2048-dimensional descriptor.

In TyreVisionX, those descriptors are treated as local patch embeddings.

## Receptive Fields
Each feature-map location sees only part of the original image, although deeper layers see larger parts. This visible area is called the receptive field.

The intuition is important:

- early layers see small edges and textures
- middle layers see larger patterns
- late layers see semantic parts and broad structure

The benchmark used ResNet50 `layer4`. That layer is powerful, but it is spatially coarse. It may be too high-level for small tyre cracks.

## Local Descriptors and Patch Embeddings
A local descriptor is the vector at one feature-map position. Instead of making one vector for the whole tyre, the patch-aware method makes many vectors:

```text
image -> feature map -> patch descriptors
```

Each descriptor represents a local tyre region. If a region contains a defect, its descriptor may be far from normal descriptors.

## Memory Bank Anomaly Detection
The memory bank is a collection of patch descriptors from normal training images only. It is the model's memory of normal tyre surface appearance.

During evaluation:

1. extract patch descriptors from a new tyre image
2. compare each patch descriptor to the normal memory bank
3. compute nearest-neighbor distance for each patch
4. aggregate patch distances into one image anomaly score

If a patch is far from all normal patches, it receives a high anomaly score.

## Light Mathematical View
Let `x_i` be a patch descriptor from a test image. Let `M` be the memory bank of normal patch descriptors.

The patch anomaly score is:

```text
score(x_i) = min distance(x_i, m), for m in M
```

The image score can be:

```text
max_i score(x_i)
```

The max rule says: if any local region is very unusual, treat the whole image as suspicious.

## PatchCore-Style Intuition
PatchCore is a family of methods based on normal patch memory. It usually stores representative local features from normal images and scores test images by nearest-neighbor distances.

This prompt implemented a lightweight version, not full PatchCore. It used:

- ResNet50 feature-map descriptors
- a reduced normal memory bank
- nearest-neighbor patch distances
- max or top-k image aggregation

This keeps the method explainable and avoids bringing in a large new framework too early.

## Why Resized Image Patches Can Fail
The previous local-feature benchmark cut out small image patches, resized them to full input size, and ran them through ResNet50. That can distort scale and context. A tiny normal rubber texture patch may look strange after being stretched to 224x224.

Feature-map patches are more principled because the image is processed once at its natural scale, and local descriptors come from inside the convolutional network.

## Why Calibration Still Matters
Anomaly scores are not labels. A threshold converts a score into a decision.

Lower threshold:

- catches more defects
- increases false positives

Higher threshold:

- reduces false positives
- may miss defects

For TyreVisionX, false negatives are critical because they mean a defective tyre was treated as normal. This is why validation-only threshold selection remains important even when the feature method changes.

## What Happened in This Benchmark
The high-recall reference remained best:

```text
resnet50_knn_threshold_sweep_reference
recall: 0.9462
false negatives: 7
false positives: 36
```

The feature-map patch methods did not beat it:

```text
featuremap patch kNN
recall: 0.1000
false negatives: 117

featuremap patch kNN threshold sweep
recall: 0.4615
false negatives: 70

patchcore-lite
recall: 0.2385
false negatives: 99
```

This means the current layer4 patch-memory setup is not useful for recall-critical tyre inspection.

## How to Interpret the Failure
This does not mean local evidence is irrelevant. It means this first implementation was not the right local representation.

Likely issues:

- layer4 is too coarse for small cracks
- nearest-neighbor distances cluster too tightly
- memory-bank sampling is simple random sampling
- patch scores are not normalized by location or layer
- max/top-k aggregation may be too noisy or too conservative depending on score distribution

## Practical TyreVisionX Takeaway
The current best anomaly candidate remains the threshold-swept pooled ResNet50 kNN model. It is not production-ready, but it is the strongest recall-oriented result so far.

Patch-aware work should continue only if the visual review shows that remaining misses are genuinely local. The next better patch-aware experiment should use layer3 or layer2+layer3 descriptors and better score normalization.

## How to Explain This in a Project Discussion
A concise explanation:

> We tested a PatchCore-style idea using ResNet50 feature-map patches because missed tyre defects may be local. The first layer4 patch-memory implementation did not improve recall; it missed many more defects than the threshold-swept pooled ResNet50 reference. This suggests that either layer4 features are too coarse for tyre defects or the patch-score calibration needs improvement. The best current candidate remains the validation-thresholded ResNet50 kNN baseline, while future patch-aware work should test mid-level features and better normalization.
