# Study Notes: Lower-Level Patch Anomaly and External Dataset Readiness

## Big Picture
TyreVisionX is trying to detect defective tyres while missing as few defects as possible. The current best recall-oriented anomaly model is not a patch model. It is a pooled ResNet50 embedding model with k-nearest-neighbor scoring and a validation-selected threshold. It catches most defects in D1, but it still misses 7 defective tyres and creates 36 false positives.

The natural research question is: are those remaining misses local tyre defects that whole-image pooling dilutes? If yes, patch-aware anomaly detection should help. The first patch-aware attempt used a high-level ResNet50 `layer4` feature map and did not help. This prompt tested a better version of the idea: use lower or mid-level feature maps and normalize patch scores more carefully.

The secondary task was external dataset readiness. Roboflow datasets may help later, but only if their provenance, license, task type, and labels are audited before import. More data is not automatically better if it contaminates the benchmark or introduces unclear labels.

## Intermediate CNN Feature Maps
A convolutional neural network does not only produce one final vector. Inside the network, each layer produces a grid of feature activations. Early layers keep more spatial detail and detect simple visual patterns such as edges or texture. Later layers are more semantic but spatially coarser.

For tyre defects, this matters because a crack or damaged patch may occupy only a small region. If the model averages the whole image into one vector, that local evidence can be diluted. A feature-map patch method tries to preserve local evidence by scoring many positions in the feature map.

In this prompt, the lower/mid-level variants used:

- `layer3`: more spatial detail than `layer4`, but still semantically useful.
- `layer2_layer3`: a combined feature map intended to mix lower texture detail with higher context.

## Receptive Field and Spatial Resolution
Every feature-map cell "sees" a region of the original image. This region is called the receptive field. In deeper layers, each cell sees a larger portion of the image, but the feature map has fewer cells. In earlier layers, each cell sees a smaller region and the map has more spatial positions.

The original patch-aware run used `layer4`. That can be too coarse for small tyre defects because each descriptor may summarize too much of the image. The layer3 and layer2+layer3 follow-up was meant to make each patch descriptor more local.

## Patch Descriptor Normalization
Patch descriptors and patch anomaly scores can have unstable scale. If one feature layer naturally produces larger distances than another, the score threshold can become hard to interpret. Robust normalization reduces this problem.

This prompt added median/MAD score normalization. The median is a robust center estimate, and MAD is a robust spread estimate. The idea is to convert raw patch scores into a more comparable scale without being dominated by a few outliers.

In TyreVisionX, this normalization is fitted only on validation-normal scores. That preserves the principle that threshold calibration should not use test labels.

## Memory Banks and Nearest-Neighbor Anomaly Scoring
A memory-bank anomaly detector stores feature descriptors from normal training images. At test time, a patch is considered normal if it is close to at least one normal patch in the memory bank. It is considered suspicious if its nearest normal neighbor is far away.

This is PatchCore-style intuition in a lightweight form:

1. Extract normal patch descriptors.
2. Store a memory bank of those descriptors.
3. Extract descriptors from a new image.
4. Score each patch by nearest-neighbor distance to the normal memory.
5. Aggregate patch scores into one image-level anomaly score.

This approach sounds ideal for local defects, but it can fail when normal tyre texture is diverse. A normal tread groove, sidewall marking, dirt spot, or lighting change may look unusual in patch space. At the same time, a subtle defect may still look close to a normal patch.

## Image-Level Score Aggregation
A patch model produces many scores per image. The system must combine them into one image score. Common choices include:

- maximum patch score,
- average of the top few patch scores,
- percentile of patch scores.

Maximum scoring is sensitive and can catch small defects, but it can also create false positives from harmless texture. Top-k averaging is slightly more stable, but it can still be fooled by repeated normal texture. In this prompt, the variants used max or top-3 mean aggregation.

## What Happened in This Benchmark
The high-recall reference stayed much stronger:

- `resnet50_knn_threshold_sweep_reference`: recall `0.9462`, false negatives `7`, false positives `36`
- `resnet50_layer3_patch_knn`: recall `0.1231`, false negatives `114`
- `resnet50_layer3_patch_knn_threshold_sweep`: recall `0.3385`, false negatives `86`
- `resnet50_layer2_layer3_patch_knn_threshold_sweep`: recall `0.2154`, false negatives `102`

This means the lower/mid-level patch memory did not separate defective tyres from good tyres. The result is not a final rejection of all patch-aware anomaly detection, but it is a strong warning that this specific implementation is not ready to replace the pooled reference.

## Why Layer3 Did Not Automatically Fix The Problem
Layer3 has more spatial detail than layer4, but more spatial detail also means more normal variation. Tyres contain many normal local structures: grooves, sidewall text, dirt, wear, lighting changes, and camera framing differences. A nearest-neighbor memory bank can treat these as anomalous even when the image is labelled good.

Also, a local tyre defect may be subtle enough that its patch descriptor remains close to normal patches. In that case, preserving spatial resolution is not enough. The model may need better features, better localization supervision, better normal-data coverage, or a different patch scoring strategy.

## Dataset Provenance and Label Harmonization
The Roboflow audit is a separate but related data-readiness task. External datasets can help, but only when their origin and labels are understood.

Provenance means knowing where data came from and under what conditions it can be used. Label harmonization means mapping external labels into TyreVisionX concepts without losing meaning. For example, one dataset may use `defective`, another may use `Defected`, and another may have bounding boxes for `cut` or `bead_damage`. These should not be blindly merged.

In this prompt, classification datasets were treated as near-term candidates only after review. Detection and segmentation datasets were marked as later-phase resources because their labels support localization rather than the current image-level anomaly benchmark.

## Why External Datasets Must Be Audited Before Use
External data can improve model coverage, but it can also damage the experiment:

- license terms may not allow the intended use,
- augmented versions may inflate image counts,
- duplicates may leak between train and test,
- labels may not match TyreVisionX definitions,
- detection datasets may not fit an image-level classification/anomaly task,
- web or public datasets may contain ambiguous examples.

For that reason, the registry is deliberately conservative. It prepares import review but does not download or merge anything.

## How To Interpret This Prompt's Result
The practical conclusion is simple: the current best TyreVisionX anomaly candidate remains the pooled ResNet50 kNN threshold-sweep model. Lower/mid-level patch-memory variants did not reduce false negatives.

The research conclusion is more nuanced. The remaining high-recall false negatives still look subtle and local enough to justify local evidence research, but the current patch-memory route needs diagnosis before more variants are added. The next useful step is to inspect patch-score distributions and nearest-neighbor patch examples, not to blindly stack more layers or backbones.

The external-data conclusion is also conservative. Good Tire Bad Tire and Tires Defects are the most plausible near-term Roboflow candidates, but they still need license citation, duplicate checks, and label audit. Tire Tread and defect by Hemant are better saved for localization. Tire Quality and the College segmentation dataset remain blocked until source/license details are verified.
