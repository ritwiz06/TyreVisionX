## Implementation Summary

- Added a reproducible generator script for the manual Google discovery pack.
- Generated 40 browser-ready manual Google image-search links grouped into five tyre-view families.
- Added damage/defect negative keywords to every query.
- Created a 40-row prefilled candidate CSV with blank source/page/local-path fields for manual completion.
- Added a manual browsing checklist and candidate-selection rubric.
- Added a professor-readable notebook that explains how to use the pack.
- Added smoke tests for query generation, CSV generation, and required metadata columns.

## Why Each Change Was Made

- The generator script keeps the markdown, CSV, and notebook in sync.
- The grouped query batches make browsing easier and support the 30-50 candidate pilot target.
- Negative keywords reduce obvious damaged/defective/irrelevant results without claiming the remaining results are normal.
- The prefilled CSV reduces user friction while preserving manual approval and provenance capture.
- The rubric makes the review criteria explicit for a beginner or professor review.
- The Pexels default was corrected because credentials must never be hardcoded.
